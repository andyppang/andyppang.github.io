Github Page绑定域名

本文最后更新于：2022年5月14日 中午

轻松两步就可以完成Github Page域名绑定

DNS端的设置

  • 增加一条 CNAME 记录，名称指向自己的 github page 页 username.github.io
类型 名称 内容
CNAME www username.github.io
  • 增加一条 A 记录，指向 185.199.108.153 或 185.199.109.153 或 185.199.110.153 或 185.199.111.153
类型 名称 内容
A @ 185.199.108.153

Github Pages端设置

（这样设置有坑，可以不看了，看下一步）

settings --> Pages --> Custom domain --> www.zeroandone.tk --> save

如果你的DNS配置正确的话，检测通过，稍等片刻，这时可以通过 http://www.zeroandone.tk 访问了

建议勾选下方的 Enforce HTTPS ，这样就可以通过 https://www.zeroandone.tk 访问

说明：zeroandone.tk 会重定向到 www.zeroandone.tk

刚把这篇文章写完，去 hexo g -d 以后，突然访问 https://www.zeroandone.tk 页面 404 了，好奇怪！

没发这篇文章之前还访问好好的，搜索发现好多小伙伴也有类似的事情发生：github page+hexo 绑定域名后页面显示404，原来是部署覆盖掉了CNAME文件，正确的处理方式看下一步

Hexo博客端设置

在 source 文件夹下创建 CNAME 文件（注意一定是大写）

文件的内容就一行 www.zeroandone.tk （换成你自己的域名）

这样部署上传的时候，就将该文件push到github仓库了
